# Slack メッセージ優先度判定

Slack メッセージの優先度を判定してください。

## 判定基準

### high (緊急・重要)
- 緊急対応が必要な障害報告、インシデント
- 明確な期限がある依頼 (今日中、本日中、ASAP、至急)
- ブロッカーとなっている問題
- 本番環境に関する問題
- セキュリティに関する報告
- 上長・顧客からの直接の依頼

### medium (通常)
- 一般的なタスク依頼・質問
- コードレビュー依頼
- ミーティング調整
- 確認依頼
- 期限が明確でない作業依頼

### low (情報共有)
- FYI (参考情報)
- 一般的なお知らせ
- 雑談
- bot からの自動通知
- 既に解決した報告

## 入力

- メッセージテキスト
- 送信者名
- チャンネル名
- メッセージタイプ (mention/channel/dm/keyword)

## 出力形式

以下の JSON 形式で出力してください:

```json
{
  "priority": "high" | "medium" | "low",
  "reason": "判定理由 (簡潔に)"
}
```

## 注意事項

- DM やメンションは優先度が高い傾向がありますが、内容で判断してください
- 日本語・英語両方に対応してください
- 迷った場合は medium としてください
